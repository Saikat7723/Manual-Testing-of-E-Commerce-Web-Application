# Manual-Testing-of-E-Commerce-Web-Application
Performed manual testing to validate core functionalities including login, product search, add to cart, and checkout workflows.
## Project Overview

This project demonstrates manual testing of an e‑commerce web
application.\
The goal of this project is to validate the core functionalities of the
system such as login, product search, cart management, and checkout
workflow.

## Testing Scope

Modules tested:

-   Login
-   Product Search
-   Product Details
-   Add to Cart
-   Checkout
-   Logout

## Testing Types Performed

-   Functional Testing
-   UI Testing
-   Regression Testing
-   Smoke Testing
-   Usability Testing

## Test Artifacts Included

The repository contains the following testing documents:

-   Test Cases (Excel)
-   Bug Report Sheet
-   Requirement Traceability Matrix (RTM)
-   Test Plan
-   Sample Test Case PDF

## How to Execute Test Cases

1.  Open the testing website (example: SauceDemo).
2.  Execute test cases from the TestCases folder.
3.  Compare actual result with expected result.
4.  Mark test case status as Pass or Fail.
5.  Log defects in Bug Report sheet if any issue is found.
